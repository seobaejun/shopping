## 2. 관리자 – 회원조회

- **가짜/테스트 데이터 제거 및 진짜 데이터만 표시**
  - 회원조회 목록에서 **테스트·가짜 회원은 제외**하고, 실제 회원만 보이도록 했습니다.
  - 제외 대상: `test_`로 시작하는 아이디, `@test.com` 이메일, 회원A~F, 테스트MD1/2, 4자리테스트/5자리테스트 등.
  - **「테스트 데이터 삭제」** 버튼을 추가해, DB에 남아 있는 테스트 회원·MD를 한 번에 삭제할 수 있게 했습니다. (삭제 후 진짜 데이터는 그대로 유지됩니다.)

- **페이지네이션 수정**
  - 전체 회원이 1페이지만 있을 때도 **페이지 번호 "1"이 보이도록** 수정했습니다.
  - 1페이지일 때 이전/다음 버튼은 비활성화된 상태로 표시됩니다.

## 3. MD 관리자 전용 수정

### 3-1. 회원조회 화면

- **표시 항목 단순화 (MD 관리자만 해당)**
  - 일반 관리자 페이지는 기존대로 두고, **MD 관리자 회원조회**만 아래 7개 항목만 보이도록 했습니다.
    - 순번 / 아이디 / 이름 / 가입일자 / 추천인 / 구매총액 / 지원토큰
  - 전화번호, 주소, 은행, 계좌번호, 상태, 관리 버튼 등은 MD 관리자 화면에서는 제거했습니다.

### 3-2. 개인별 구매 누적정보

- **전화번호 비노출 (MD 관리자만 해당)**
  - 개인별 구매 누적정보의 **회원 정보** 카드에서 **전화번호**를 MD 관리자 화면에서만 숨기도록 했습니다.
  - 일반 관리자 페이지에서는 전화번호가 그대로 표시됩니다.

### 3-3. 4자리 / 5자리 권한 로직 (기존 구현 확인)

- **회원조회·개인별 구매 누적정보** 모두 아래 로직으로 동작하는 것을 확인했습니다.
  - **4자리 MD (예: 1234):**  
    본인 코드(1234) + 하위 5자리(12341~12349) 추천인으로 가입한 회원만 조회·검색 가능
  - **5자리 MD (예: 12345):**  
    해당 5자리(12345) 추천인으로 가입한 회원만 조회·검색 가능
- 추천인 ID 검색도 위 범위 안에서만 동작합니다. (4자리는 1234~12349 구간 검색, 5자리는 사실상 본인 코드만 해당)

## 4. 가로 메뉴(네비) 서브메뉴가 안 열리던 이유 & 수정

### 왜 이전에는 안 열렸는지

1. **이벤트가 우리 코드까지 오지 않음**  
   가로 메뉴는 Firestore에서 카테고리를 불러온 **뒤에** 비동기로 HTML이 채워집니다.  
   처음에는 `<a href="#">` + 인라인 `onclick` 또는 `body`에 단순 클릭 리스너만 걸어 두었는데,  
   - 다른 스크립트가 먼저 클릭을 막거나(`preventDefault`/`stopPropagation`)  
   - 모바일에서 `touchend`와 `click`이 겹치면서 한 번만 처리되거나  
   - 인라인 `onclick`이 CSP 등으로 실행되지 않는 환경이 있으면  
   우리 핸들러가 실행되지 않아 “눌러도 반응 없음”이 됐습니다.

2. **리스너를 붙이는 타이밍**  
   `DOMContentLoaded`에서만 리스너를 붙이면, 그 시점에는 아직 메뉴 HTML이 비어 있을 수 있어서,  
   “어디에 클릭이 왔는지” 판단하는 위임 대상이 불안정할 수 있었습니다.

3. **마지막으로 적용한 해결**  
   - 하위가 있는 항목을 **`<a>` 대신 `<span data-mainnav-toggle>`** 로 바꿔서, 링크 기본 동작과 겹치지 않게 했고  
   - **`document.body`에 `capture: true`** 로 클릭/터치/키보드 리스너를 걸어, 다른 스크립트보다 **먼저** 처리되게 했고  
   - 서브메뉴가 열릴 때 **navbar `.container`에 `overflow: visible`** 을 주어, 드롭다운이 잘리지 않게 했습니다.

### 오른쪽으로 넘긴 뒤 열 때 왼쪽 콘텐츠를 덮던 현상

- 가로 스크롤로 오른쪽 메뉴(예: 반려동물)를 누르면, 서브메뉴가 해당 `li` 기준 `left: 0`으로 붙어 있어서  
  화면 기준으로는 **왼쪽(히어로 배너 등)을 덮는** 형태로 보였습니다.
- **수정:** 서브메뉴를 열 때, 해당 `li`의 `getBoundingClientRect().left`가 0보다 작으면  
  서브메뉴의 `left`를 그만큼 오른쪽으로 밀어서, **뷰포트 왼쪽 밖으로 나가지 않도록** 보정했습니다.  
  → 오른쪽으로 넘겨서 연 메뉴도 이제 왼쪽 본문을 덮지 않고, 화면 안에 맞게 보입니다.
