# 이미지 최적화 / 리사이징 옵션

## 지금 적용된 것 (프론트만)

- **메인 이미지**: `decoding="async"` + `fetchpriority="high"` → 메인 스레드 덜 쓰고, 대표 이미지 먼저 로드
- **상세 이미지**: `loading="lazy"` + `decoding="async"` → 화면에 가까워질 때 로드, 디코딩 비동기

용량/해상도 변경 없이 **로딩 순서와 디코딩**만 최적화된 상태입니다.

---

## 리사이징·포맷 최적화를 하려면 (선택)

### 1. 이미지 서버에서 리사이즈 지원

- 예: `http://1.228.243.28/item/xxx.png?w=800` 처럼 쿼리로 너비 지정
- 지원한다면 `image-config.js`에서 `resolveProductImageUrl`이 `?w=800` 등을 붙이도록 수정 가능
- 상세 페이지는 `srcset`으로 400/800/1200 등 여러 크기 요청 가능

### 2. 이미지 CDN (Cloudinary, imgix 등)

- 원본 URL을 CDN 주소로 감싸서 리사이즈·WebP/AVIF 자동 적용
- 예: `https://res.cloudinary.com/xxx/image/fetch/w_800,f_auto/https://...`
- 계정/도메인 설정 후 `resolveProductImageUrl`에서 이 형식으로 바꿔주면 됨

### 3. Vercel Image Optimization

- Vercel 플랜에 따라 `/_vercel/image?url=...&w=800&q=80` 형태 사용 가능
- 정적 HTML 프로젝트에서는 공식 지원이 제한적이라, 사용 가능 여부는 배포 환경에서 확인 필요
- 가능하다면 이미지 URL을 위 쿼리 형태로 바꿔서 리사이즈·포맷 최적화 적용 가능

### 4. 별도 리사이즈 API (서버리스 등)

- Vercel/Netlify 함수 등에서 Sharp 등으로 리사이즈 후 반환
- 예: `/api/image?url=...&w=800` → 서버가 이미지 받아서 800px로 리사이즈 후 전달
- 구현 난이도와 비용이 있어서, CDN이나 이미지 서버 지원이 없을 때 검토

---

정리: **지금은 프론트 최적화만 적용됨.** 실제 **용량/해상도 줄이기**는 위 1~4 중 하나(이미지 서버 옵션, CDN, Vercel, API)를 선택해 구현하면 됩니다.
