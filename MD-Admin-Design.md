# MD 관리자페이지 설계서

## 개요
- MD(사업자)가 자신의 추천인 코드로 가입한 회원들의 매출을 조회할 수 있는 관리 페이지
- 기존 관리자페이지의 사이드바 형식을 참고하여 구성

## 추천인 코드 체계
- **4자리 코드**: 상위 MD (예: 1024)
- **5자리 코드**: 하위 MD (예: 12041, 12042, 12043, 12044)
- **조회 규칙**:
  - 4자리 코드 조회 시: 해당 코드 + 하위 5자리 코드 모든 매출 조회
  - 5자리 코드 조회 시: 해당 5자리 코드 본인 매출만 조회

## 페이지 구조

### 1. 사이드바 메뉴
```
MD 관리자
├── 대시보드
│   └── 매출 현황 요약
├── 매출 관리
│   ├── 추천인별 매출 조회
│   └── 회원별 매출 상세
└── 회원 관리
    ├── 추천 회원 목록
    └── 회원 상세 정보
```

### 2. 메인 기능

#### 2.1 추천인별 매출 조회
- **입력**: MD 코드 (4자리 또는 5자리)
- **출력 테이블**:
  | 순번 | 아이디 | 이름 | 가입일자 | 추천인 | 구매총액 | 지원토큰 |
  |------|--------|------|----------|---------|----------|----------|
  | 1    | user01 | 홍길동 | 2024-01-15 | 12041 | 150,000원 | 1,500 trix |
  | 2    | user02 | 김철수 | 2024-01-20 | 12042 | 200,000원 | 2,000 trix |

#### 2.2 필터링 기능
- 기간별 조회 (시작일 ~ 종료일)
- 추천인 코드별 필터링
- 매출액 범위 설정

#### 2.3 통계 정보
- 총 추천 회원 수
- 총 매출액
- 총 지원토큰
- 평균 구매액

## 기술 구현

### 1. 데이터 소스
- Firestore `members` 컬렉션: 회원 정보 (mdCode 필드 포함)
- Firestore `orders` 컬렉션: 주문/매출 정보

### 2. 조회 로직
```javascript
// 4자리 코드 조회 (1024)
function getSubordinateSales(mdCode) {
    // mdCode가 1024인 회원들 조회
    // mdCode가 12041, 12042, 12043, 12044... 인 회원들 조회
    // 모든 해당 회원들의 주문 내역 합산
}

// 5자리 코드 조회 (12041)
function getPersonalSales(mdCode) {
    // mdCode가 정확히 12041인 회원들만 조회
    // 해당 회원들의 주문 내역만 조회
}
```

### 3. 보안
- MD 로그인 시스템 (별도 인증)
- 자신의 코드 하위만 조회 가능하도록 제한
- 세션 관리
